MiU API Lifecycle

MiU icinde bir API istegi geldiginde public endpoint, protected endpoint, token dogrulama, rate limit, validation ve permission kontrollerinin nasil zincir halinde calistigini gosteren gorsel ozet.

Genel API Akisi

Login Akisi

Public Endpoint

/api/v1/auth/login public action olarak isaretlidir. Bu yuzden ilk login isteginde token beklenmez.

Validation

AuthLoginRequest email, password ve varsa device_name alanlarini kontrol eder.

IdentityService

loginUserForApi() once kullanici sifresini dogrular, sonra yeni token olusturur ve api_tokens tablosuna kaydeder.

Token TTL

Token olusurken expires_at = simdi + token_ttl yazilir. Protected endpoint her kullanildiginda sure uzatilir; gecmisse 401 doner.

Ornek Akislar

1. Login Istegi

Ilk giriste token yoktur, sadece login endpoint cagrilir.

POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "123456",
  "device_name": "iPhone"
}

2. Protected Istek

Login sonrasi gelen token header ile gonderilir.

GET /api/v1/users
X-Api-Token: kullanicidan-donen-token

3. Permission Mantigi

Controller bazli yetkiler router tarafinda merkezi kontrol edilir.

public static function actionPermissions(): array
{
    return [
        'index' => 'users.view',
        'create' => 'users.edit',
        'delete' => 'users.delete',
    ];
}
